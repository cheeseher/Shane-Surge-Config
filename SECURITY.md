# 安全说明

本仓库是公开的 Surge Managed Profile 分发仓库。

## 禁止提交的内容

- 机场订阅 URL、订阅 token、用户 ID 或带凭据的二维码。
- 节点服务器、端口、密码、证书、私钥或 WireGuard 密钥。
- Surge Linked Profile、本地覆写配置、请求日志或可识别个人的信息。
- 带地址栏、订阅详情或账户信息的未打码截图。

真实 Amy 非托管 URL 只能保存在各设备的 Surge Linked Profile 中。仓库中的 `AMY_SUBSCRIPTION_URL` 必须始终保持为无效占位地址。

## 发现泄露后的处理

1. 不要只删除最新文件或提交；Git 历史仍可能保留内容。
2. 立即到服务商处重置订阅凭据，使旧链接失效。
3. 将仓库临时停止分发，并检查提交历史、GitHub Actions 日志和 Pages 构建产物。
4. 如需报告问题，请只描述文件路径和提交号，不要在公开 Issue 中粘贴凭据原文。

## 发布前检查

每次发布必须通过 Surge 配置检查、外部资源可访问性检查和敏感信息扫描。Telegram 与 Zoom 的真实网络质量仍需在中国大陆网络进行通话验收。
